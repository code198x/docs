# Alternative Input Devices & Peripherals

**Purpose:** Document the variety of input methods beyond joysticks and keyboards that shaped game design.

**Last Updated:** 2026-01-18

---

## Why Input Devices Matter

Different inputs created different games:
- **Light guns** → shooting galleries, Duck Hunt
- **Trackballs** → Marble Madness, Centipede
- **Steering wheels** → realistic racing
- **Paddle controllers** → Pong, Breakout, Arkanoid

Understanding input devices = understanding game design constraints and possibilities.

## Part 1: Light Guns

### How They Work

| Technology | Era | Method |
|------------|-----|--------|
| **CRT-based** | 1970s-2000s | Detect screen phosphor flash |
| **Sensor bar** | 2000s+ | IR positioning (Wii) |
| **Camera-based** | Modern | Track screen position |

**CRT light guns** only worked on tube TVs - they detected the electron beam timing to determine aim point. This is why old light guns don't work on modern displays.

## Part 2: Trackballs

### Why Trackballs

- **Precise analogue control** - Finer than joysticks
- **Momentum** - Ball keeps spinning
- **360° movement** - No stick centering

|----------------|--------|

## Part 6: Light Pens

### The Forgotten Input

Light pens were **drawing devices** that detected CRT beam position.

| Use | Platform | Example |
|-----|----------|---------|
| **Drawing** | C64, BBC | Art packages |
| **Selection** | Various | Menu navigation |
| **Gaming** | Limited | Some puzzle games |

### Key Light Pen Software

| Software | Platform | Use |
|----------|----------|-----|
| **Koala Painter** | C64 | Could use light pen |
| **Tech 2** | BBC Micro | Drawing |
| **Gibson Light Pen** | C64 | Art package |

### Why They Faded

- Arm fatigue (holding at screen)
- CRT-only technology
- Mouse/tablet proved better

## Part 9: Input Design Principles

### Matching Input to Game

| Game Type | Best Input | Why |
|-----------|------------|-----|
| **Precision shooter** | Mouse/light gun | Point accuracy |
| **Racing** | Wheel/analogue stick | Gradual steering |
| **Platformer** | D-pad/digital | Precise timing |
| **Flight sim** | Flight stick | Multiple axes |
| **Rhythm** | Dedicated controller | Physical mapping |
| **Fighting** | Arcade stick | Fast inputs |

### Teaching in Curriculum

Our platforms have specific inputs:
- **C64/Spectrum:** Joystick (1 button) or keyboard
- **NES:** D-pad + A/B + Start/Select
- **Amiga:** Joystick (1-2 buttons) or mouse

Understanding input constraints shapes game design:
- One button = jump OR fire (not both)
- No analogue = 8-way digital movement
- Keyboard = more options but less intuitive
