## Part 1: Puzzle Games

### The Tetris Phenomenon

| Aspect | Details |
|--------|---------|
| **Creator** | Alexey Pajitnov |
| **Year** | 1984 (Soviet Union) |
| **Platform** | Electronika 60, then everywhere |
| **License** | Notoriously complicated |
| **Impact** | Defined puzzle genre |

**Tetris** proved games didn't need violence, characters, or narrative - pure abstract gameplay sufficed.

### The Tetris License Wars

| Party | Claim |
|-------|-------|
| **Pajitnov** | Creator |
| **ELORG** | Soviet state agency |
| **Spectrum HoloByte** | Early US license |
| **Bullet-Proof Software** | Henk Rogers' company |
| **Mirrorsoft/Maxwell** | Disputed license |
| **Nintendo** | Eventually won Game Boy rights |

**Henk Rogers** secured Game Boy Tetris for Nintendo - defining the handheld.

### Classic Puzzle Games

| Game | Year | Innovation |
|------|------|------------|
| **Tetris** | 1984 | Falling block |
| **Columns** | 1989 | Match-3 variant |
| **Dr. Mario** | 1990 | Virus matching |
| **Lemmings** | 1991 | Action puzzle |
| **Puyo Puyo** | 1991 | Chain combos |
| **Panel de Pon** | 1995 | Swap matching |
| **Puzzle Bobble** | 1994 | Bubble shooting |

### Puzzle Game Design Principles

| Principle | Example |
|-----------|---------|
| **Simple rules** | Tetris: shapes, lines |
| **Emergent depth** | Combo strategies |
| **One more go** | Short play loops |
| **Escalating difficulty** | Speed increase |
| **Flow state** | Meditative play |

## Part 6: Design Principles

### Puzzle Game Design

| Principle | Implementation |
|-----------|----------------|
| **Teach through play** | Early levels as tutorials |
| **Fair difficulty** | Player should see solution |
| **Elegant rules** | Simple to learn |
| **Emergent depth** | Mastery takes time |

### Strategy Game Design

| Principle | Implementation |
|-----------|----------------|
| **Meaningful choices** | Decisions matter |
| **Risk/reward** | Balance gambling |
| **Information** | What to reveal |
| **Pacing** | Build to climax |

### Simulation Design

| Principle | Implementation |
|-----------|----------------|
| **Complexity management** | Hide unnecessary detail |
| **Feedback loops** | Show cause and effect |
| **Sandbox freedom** | Player sets goals |
| **Authenticity** | Feel real, not be real |
