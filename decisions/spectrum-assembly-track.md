# Decision: Spectrum Assembly Track Structure

> **AMENDED 2026-05-29 by [spectrum-assembly-gentle-ramp.md](spectrum-assembly-gentle-ramp.md).**
> The track now opens with a Primer ("Meet the Machine") and a tiny first game *ahead* of
> Shadowkeep, and obeys a **technique budget** — at most 1–2 major new techniques per game;
> split or insert where the lineup over-packs. Shadowkeep no longer introduces the full Trunk:
> the advanced rendering Trunk (pre-shift → masking → Y-sort + iso) spreads across the opening
> into Greypeak. Ordering is **hybrid** — the early sequence (opening → Greypeak) is strictly
> monotonic; later volumes assume a competent learner and ramp within-theme. The 44-entry count
> grows by the opening + rendering inserts; the gentle-ramp decision holds the revised lineup.

## The decision

The Spectrum Assembly track is a **44-entry curriculum across 6 volumes**, organized by a layered model that composes additional content onto a native L0 core:

- **L0 (Core, 48K native):** 36 games across Volumes 1-4 + 4 demoscene productions in Volume 5 = 40 entries
- **L+1 (128K Audio enhancement):** Branch G techniques (AY-3-8910 + 128K banking) — retroactive on selected L0 games
- **L+3 (Demoscene polish layer):** Selective application of Volume 5 techniques to L0 title sequences and transitions
- **L+4a (Selective Next-enhanced variants):** ~6 L0 games get classic-vs-Next-hardware comparison variants
- **L+4b (Next-native productions):** 4 entries in Volume 6 designed specifically for Spectrum Next hardware

October 2026 launch (Shadowkeep Arcs 1-2 at 32 units) is **unchanged** by this decision. All v0.1 / v0.2 / v0.3 reframe work is post-October.

## Why

Before this decision, Spectrum Assembly existed as a single committed game (Shadowkeep, 32 units, two arcs) with no surrounding track-level structure — *anaemic at the track level* compared to BASIC v6.3's 32-game / 4-volume structure. The track's pedagogical claims (commercial bar, legends technique-by-technique, constraint as design language) had no track-level structure to anchor them to.

This decision provides that structure. Each game owns a distinct design concept and a named legend tradition. Each volume has a thematic identity. Cross-cutting axes (Embedded Design Concepts, Engineering Practices, Production Craft) capture the disciplines that weave through games. The layer model honours both native Spectrum and modern Spectrum scene (Next).

## North Star hierarchy

Four claims with explicit roles:

- **Spine (outcome):** Commercial bar. *"You can ship a game CRASH would have reviewed."*
- **Method (journey):** Legends technique-by-technique. Each game pegged to a named legend (Smith / Ultimate / Crow / Hewson / Ritman / Singleton / Priestley / Crammond / Evans).
- **Lens (framing):** Constraint as design language. 48K / 8×8 attribute / single-channel beeper limits ARE the design vocabulary.
- **Depth (level of teaching):** The machine at the metal. Every claim grounded in real bytes.

Method/lens/depth all serve the spine.

## Three-axis design pattern

Each game has three orthogonal axes:

1. **Player-facing headline** — one-word player-felt experience (Atmosphere, Jump, Density)
2. **Design concept** — generalizable design idea (Sense of Place, Platform Physics, Single-Screen Composition)
3. **Technique** — engineering implementation, mapped separately in cross-game tech-tree (Y-Sort Rendering, Gravity Physics, Hand-Composed Hazard Layout)

This matches BASIC v6.3's dual-name with an explicit third (technique) axis Assembly needs because engineering is heavier in Assembly than in BASIC.

## Three cross-cutting axes

| Axis | Items | What it teaches | Applies to |
|------|-------|-----------------|------------|
| Embedded Design Concepts | ~10 | The WHY (atmosphere through restraint, authored level design, difficulty as design, etc.) | Both BASIC and Assembly |
| Engineering Practices | 9 | The HOW-ENGINEERING (Z80 optimization disciplines — SMC, T-state counting, LDIR, stack-as-data, etc.) | **Assembly only** — real differentiator from BASIC |
| Production Craft | ~15 | The HOW-PIPELINE (art tools, music trackers, loading screens, custom loaders) | Both, more pronounced in Assembly |

## Tech-tree as engine of progression

Techniques cluster into a **Trunk + Branches** model:

- **Trunk (Layers 1-4)** — introduced *gently across the opening* (Primer → tiny game → Smooth Motion → Overlap → Greypeak), not front-loaded into Shadowkeep; every later game inherits. See [spectrum-assembly-gentle-ramp.md](spectrum-assembly-gentle-ramp.md)
- **Branches A-G** — genre-conditional specialisations
  - A: Adventure Depth — Shadowkeep Arcs 3-4 in the flick-adventure tradition: items (inventory + effects), world (multiple keeps + outdoor traversal), persistence (tape save/load), secrets and lore, atmospheric cycles, collection-based endings. Per [shadowkeep-four-arc-framing.md](shadowkeep-four-arc-framing.md). **Does NOT include combat / classes / XP / spells / branching narrative / multi-NPC dialogue / full Singleton-class location-graph — those are primary teaching in V1.4 / V1.6 / V1.7 / V3.17 / V3.20 per the genre-honesty redistribution.**
  - B: Platformer Specifics (V2)
  - C: Scrolling and Density (V3)
  - D: Real-World Sim (V4 sport / vehicle)
  - E: True 3D (V4 3D entries)
  - F: Cross-cutting Specials (parser, isometric, procedural, etc.)
  - G: Audio Hardware (AY + 128K — V2.16 introduces, V3.17 deepens)

Deprecation pairs are first-class — when a richer technique replaces a simpler one, this is a *teaching moment*, not a refactor. These pairs now play out *across games* — the simple version shipped in an earlier game, the upgrade motivated by its felt limitation in a later one: cell-snapped movement (tiny game) → pre-shifted (Smooth Motion); single-draw (Shadowkeep) → masked (Overlap) → Y-sort (Greypeak); attribute-collision → tile-map-collision. Per [spectrum-assembly-gentle-ramp.md](spectrum-assembly-gentle-ramp.md).

## Layer model

| Layer | Content | Status |
|-------|---------|--------|
| L0 | 40 native entries (V1-V5) | Core curriculum |
| L+1 | 128K AY audio enhancement on selected games | Additive |
| L+3 | Demoscene polish on selected title sequences | Additive |
| L+4a | Next-enhanced variants of 6 L0 games (V1.2, V2.10, V3.17, V3.18, V4.29, V4.30) | Late curriculum extension |
| L+4b | Volume 6 — 4 Next-native productions | New volume |

L+2 (Production Polish) was originally proposed but dissolved into the Production Craft cross-cutting axis.

## Cut hierarchy

If timeline tightens post-October, cuts work backwards:

1. **V6 (L+4b)** drops first — Next-native is late-curriculum bonus
2. **L+4a Next variants** drop next — selective enhancements
3. **V5 (demoscene)** drops next — adjacent tradition
4. **L+1 retroactive AY enhancements** drop — beeper-only V1 games still work
5. **Volume 1-4 Tier 2 games (17-36, in approximate reverse importance)** drop next
6. **Volume 1-4 Tier 1 games (1-16)** are the spine; cut last

Cuts apply *only* after a real timeline slip is identified. The default plan is the full 44.

## Drift triggers

- **Extending past 32 units within October for Shadowkeep.** Still the binding drift trigger from [shadowkeep-32-unit-commitment.md](shadowkeep-32-unit-commitment.md). v0.3 / v0.2 / v0.1 additions are all post-October.
- **Adding games beyond the 44** without proposing a new track decision. Track structure is bounded; growth requires explicit proposal.
- **Treating L+4 (Next) as a separate track** rather than additive layers. Next is a layer, not a track-replacement.
- **Cloning canonical games** rather than designing inspired-by works (see [inspired-by-not-clones-naming.md](inspired-by-not-clones-naming.md)).
- **Mixing axis content** — design concepts in the technique slot, techniques in the design slot. The three-axis pattern requires clean separation.
- **Treating Engineering Practices as ad-hoc folklore** rather than named cross-cutting axis content.
- **Exceeding the technique budget without a split or insert.** A game quietly growing to 3–4 major new techniques is the failure mode the gentle-ramp decision exists to prevent. Split it or insert a game ahead of it.
- **Re-loading the first game with engine technique.** The opening (Primer + tiny game) deliberately carries the assembly-onboarding, not the engine course. If the tiny game or early Shadowkeep reaches for pre-shift / masking / Y-sort, that's the cliff returning — stop.
