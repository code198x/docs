Unit 03 · the lesson

Reading the keyboard

INKEY$ is going to surprise you.

~20 min Foundation Sinclair BASIC

What you'll build

By the end of this lesson your program will know which number the player typed, and react to it. We start with the simplest way to ask a question — and then meet the reason that simple way won't survive contact with a real-time game.

Getting started

Type in the listing below and run it. Type a number. Type another number. The line that does most of the work is line 30 — hold that thought.

lucky-number.basSINCLAIR BASIC
10REM lucky number
20LET n=INT(RND*100)+1
30INPUT "Your guess: ";g
40IF g=n THEN GO TO 100
50IF g<n THEN PRINT "Too low"
60IF g>n THEN PRINT "Too high"
70GO TO 30
100PRINT "Got it!": BEEP 1,12
Two ways to ask. One waits, one doesn't — that's the whole game.

INPUT halts everything. While you're typing, the whole program waits — the interpreter draws a flashing cursor and does nothing else. That's fine for a guessing game; there's nothing to do while the player thinks. But the next game we build, Reflex, has to read the keyboard without waiting. For that you need INKEY$. We'll reach for it next unit.

Try this
  • Type 200. What happens — and is that a bug?
  • Type hello. Why does it error?
  • Remove line 30, then run it. What did line 30 actually do?
