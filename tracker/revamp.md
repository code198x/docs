## The principle that governs everything

**Quality bar is non-negotiable.** If runway runs short, we ship *fewer items at full quality*, never *more items at lower quality*. Cuts are to the count, never to the bar.

This is the whole point of the project. Doing it half-arsed loses the reason to be doing it at all.

## Locked decisions across the project

- **Bar (Assembly tracks):** mid-tier full-price at the 256-unit game endpoint — Hewson / Gremlin / Mikro-Gen / Durell era, ~£7.95 in period. Multi-disciplinary (code + visuals + audio + level + polish co-equal). See [decisions/real-retro-games.md](../decisions/real-retro-games.md) and [decisions/curriculum-structure.md](../decisions/curriculum-structure.md).
- **Bar (BASIC track):** Usborne 1984. On-ramp confidence-builder. *Computer Spacegames* not *The Hobbit*. Kids, beginners, returners. Short programs, type-it-in, see-it-work.
- **Voice (site-wide):** magazine-class, not academic. *Crash* / *Zzap!64* / *Your Sinclair* / *ACE*. Warm, technical, opinionated, never condescending. See [writing-voice.md](writing-voice.md).
- **Unit counts: powers of 2, nested.**
  - Per-game: **8, 16, 32, 64, 128, 256, 512.** 8 default; 16 common; 32+ needs design justification; the bar climbs steeply with the number.
  - Platform totals: **up to 1024, 2048, 4096, even 8192** — these are reserved for full-platform sums (every game, every phase, every unit). Aspirational, not forced. Per-game discipline naturally rolls up; if the platform total falls cleanly on a power of 2, take it; if not, don't pad.
  - Don't pad to hit a number. Let the design dictate.
- **Pattern (Cadence / Nightshade / Exodus):** type-in scaffold (Lucky Number style). Black boxes open progressively.
- **Pattern (Shadowkeep):** **incremental scaffold within phases, spiral progression across phases.** Starts from nothing, each unit adds one feature; phase boundaries are spiral revisit points where earlier work is deepened (animation revisits movement, scale revisits room engine, etc.). The Spectrum's foundation is approachable enough that no pre-typed engine is justified. Deliberate divergence from the C64/NES/Amiga pattern. See [decisions/spiral-and-incremental.md](../decisions/spiral-and-incremental.md).
- **Source of truth for scope:** **this tracker.** Spec drifts because nobody reads it; tracker is what gets opened each session.
- **Signal:** dropped, replacement TBD post-Phase 1.

## Definition of Done (per unit)

A unit isn't shippable until all nine tick:

- [ ] MDX written (magazine voice, length appropriate to the unit's teaching beat)
- [ ] Code listing assembles cleanly with the platform's tools
- [ ] Code listing runs correctly on the platform's emulator (Fuse for Spectrum, etc.)
- [ ] Code listing runs correctly on real hardware (Spectrum Next for Spectrum)
- [ ] At least one screenshot captured
- [ ] All `<CodeFromFile>` snippets exist in code-samples and assemble
- [ ] "Try this" experiments tested and produce the described result
- [ ] "If it doesn't work" troubleshooting reflects actual failure modes encountered during testing
- [ ] **All cross-references resolve to real content** — vault entries, pattern library entries, and other units linked from this unit must exist (or be catalogued as required, see [lesson-references.md](lesson-references.md))

## Cross-referencing discipline

Every unit should weave in links to relevant **vault entries** (games, hardware, people, techniques, culture) and **pattern library entries** as it teaches. This is what turns a curriculum from a tutorial into a living encyclopedia of 80s computing — the reader can click through from a Shadowkeep unit to *Atic Atac*'s vault entry to Ultimate Play the Game's company page in one read.

**Catalogue references as we go.** Each new unit's references are logged in [lesson-references.md](lesson-references.md) as it's written. The catalogue does two jobs:

1. **Drives vault completeness work.** If a unit references `/vault/games/atic-atac` and the page doesn't exist, the catalogue surfaces the gap.
2. **Surfaces reuse.** If five units reference *attribute-clash*, that vault entry deserves real care.

The catalogue is the operational link between lesson work and vault work.

## Cadence (C64) — DEFERRED

**Status:** Phase 1 spec + Unit 1 MDX drafted, deferred until post-October to clear runway for Shadowkeep.

**Genre:** Horizontal music-driven shoot-em-up.
**Setting:** The Resonance Belt — sound-as-physical-force sci-fi. Player pilots the sound-ship *Cadence*. Discord enemies are broken music.
**Visual direction:** Music made visible — oscilloscope waveforms as terrain, frequency-bar columns, harmonic ripples. Black + cyan + magenta + white + light blue + dark grey.
**Length:** ~2:30 song = level. The song IS the level. Surviving the song wins.

### 16-unit spine (locked)

| # | Title | What ships |
|---|---|---|
| 1 | Boot | Typed scaffold: ship + 3-voice music + horizontal scroll + raster IRQ |
| 2 | Steer | Joystick movement, screen bounds |
| 3 | Fire | Bullets travel right |
| 4 | Beat | Music tick exposed; first enemies spawn on beat |
| 5 | Voice 1 | Music driver opens (1/2) — voice 1 internals |
| 6 | Patterns | Enemy movement patterns (descent, sine, dive) |
| 7 | Hit | Collision, hit flash, voice 3 SFX |
| 8 | Score | BCD score, screen RAM display |
| 9 | Three Voices | Music driver opens (2/2) — full arrangement |
| 10 | Multitudes | Indexed enemy table, formations |
| 11 | Lives | Death, respawn, invulnerability |
| 12 | Scroll Engine | Hardware scroll opens — `$D016` + screen RAM column shifts |
| 13 | The Bridge | Boss fight at song's bridge; scroll freezes |
| 14 | States | Title, win (survive song), lose |
| 15 | Lustre | Parallax (raster split), border-pulse, screen shake |
| 16 | Phase 1 Complete | Polish, high score, integration, credits |

### Music composition

**Composer:** Steve (with fallback to paid composer for sections that don't land).

**Format:** 3 streams of `(note, duration)` byte pairs per voice.

**Notes:** `$00` = rest, `$01`-`$5D` = C0-B7. Standard chromatic, scaled to SID's 8-octave range.

**Durations** (in driver ticks at 50Hz):
- 9 = 16th note at 130 BPM
- 18 = 8th
- 36 = quarter
- 72 = half
- 144 = whole

**Loop marker:** `!byte $FF, $FF` returns the voice pointer to the start of its data.

**Voice roles:**
- V1 — Lead melody, pulse waveform with PWM, fast attack
- V2 — Bass + harmony, triangle, walking bass approach
- V3 — Percussion (noise) **shared with SFX** — game sounds preempt drums on collisions/fire

**Tempo / key:** 130 BPM / A minor (overridable).

**Song structure:** Intro 16 bars / Verse 1 32 / Chorus 16 / Verse 2 32 / Bridge (boss) 32 / Finale 16.

**References:** Hubbard *Commando*, *Sanxion* loader, Galway *Times of Lore*, Tel Jeroen *Wizball*, *Otocky* soundtrack.

**Side product:** composition process publishable as "Composing for SID" series. Cadence soundtrack as worked example.

## Exodus (Amiga)

**Genre:** Lost-Vikings-flavoured 3-creature puzzle (NOT Lemmings-clone).
**Title kept:** *Exodus* — fits "small party escapes" better than the Lemmings reading.
**Era reference:** 1991 Psygnosis / Mirrorsoft budget release.

### Three creatures (concept)

| Creature | Verb | Unique ability |
|---|---|---|
| Climber | *Reach* | Climb walls, fit narrow gaps, jump higher |
| Bruiser | *Break* | Smash terrain, push blocks, kill blocking enemies |
| Engineer | *Use* | Pick up + place items, operate mechanisms |

Direct control + switching (1/2/3 keys). Item-based puzzle vocabulary. Single multi-screen level by Phase 1 end with all three required.

### Concept history

Original concept by Steve in school (~1990, working name "Ellorrs" — not kept). Pre-dates Lost Vikings (1992); parallel invention.
