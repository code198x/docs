| Company | Active | Notable People | Key Games |
|---------|--------|----------------|-----------|
| Imagine Software | 1982-84 | Mark Butler, Eugene Evans | Spectacular collapse |
| Ocean Software | 1983-96 | Gary Bracey, Jon Woods | Licensed games empire |
| Psygnosis | 1984-93 | Ian Hetherington | Shadow of the Beast, Lemmings |
| Software Projects | 1983-88 | Matthew Smith | Manic Miner, Jet Set Willy |
| Bug-Byte | 1980-85 | | Early Spectrum games |

**Why Liverpool?** Strong art school tradition, affordable rent, clustering effect once Imagine started. Ocean's success created a talent pool that Psygnosis drew from.

## Part 4: Techniques - Vintage to Modern

How skills learned on vintage hardware apply to modern development.

### Memory & Performance

| Vintage Technique | Modern Application | Why It Matters |
|-------------------|-------------------|----------------|
| **Fixed-point arithmetic** | Deterministic netcode (fighting games, RTS) | Floating point is non-deterministic across platforms |
| | Embedded systems, IoT | Many devices lack FPU |
| | Financial calculations | Avoiding rounding errors |
| | FPGA/hardware design | Fixed-point is hardware-native |
| **Object pooling** | Unity/Unreal pooling systems | Avoiding garbage collection stalls |
| | Mobile performance | Memory allocation is expensive |
| | VR (90fps requirement) | No frame drops allowed |
| **Memory-conscious design** | Mobile development | RAM constraints still exist |
| | WebGL/browser games | Limited resources |
| | Embedded/console | Fixed memory budgets |
| | Streaming worlds | Managing loaded assets |

### Graphics & Rendering

| Vintage Technique | Modern Application | Why It Matters |
|-------------------|-------------------|----------------|
| **Tile-based rendering** | Unity Tilemap, Godot TileMap | 2D games still use tiles |
| | GPU tile-based deferred rendering | Mobile GPUs (PowerVR, Mali, Adreno) |
| | Procedural generation | Tiles as building blocks |
| **Sprite sheets / atlases** | Texture atlases | Reducing draw calls |
| | UI sprite packing | Same optimization principle |
| **Double/triple buffering** | VSync, frame pacing | Tear-free rendering |
| | Vulkan/Metal/DX12 | Explicit buffer management |
| **Raster effects / per-scanline** | Shader programming | Per-pixel operations |
| | Post-processing | Same thinking, different hardware |
| **Copper lists / display lists** | GPU command buffers | Vulkan/Metal command lists |
| | Deferred rendering | Queueing render operations |
| **Palette manipulation** | Colour grading | Real-time colour transforms |
| | Shader colour effects | LUTs (Look-Up Tables) |
| **Y-depth sorting** | 2D games in 3D engines | Sprite ordering |
| | Transparency sorting | Same problem, modern context |

### Data Structures & Algorithms

| Vintage Technique | Modern Application | Why It Matters |
|-------------------|-------------------|----------------|
| **Ring buffers** | Audio processing | Circular buffers everywhere |
| | Network packet handling | Streaming data |
| | Input replay / undo | Position history |
| | Lock-free queues | Concurrent programming |
| **State machines** | Animation state machines | Unity Animator, Unreal ABP |
| | AI behavior | Foundation of behavior trees |
| | UI flow | Screen management |
| | Game states | Still the standard pattern |
| **Lookup tables** | Shader constants | Pre-computed values |
| | Animation curves | Baked calculations |
| | Audio wavetables | Synthesis |
| **Bit manipulation** | Shader programming | Packing data |
| | Network protocols | Efficient encoding |
| | Flags and state | Boolean packing |
| **Spatial partitioning** | Physics engines | Broad phase collision |
| | Culling systems | Visibility determination |
| | AI navigation | Pathfinding grids |

### Audio

| Vintage Technique | Modern Application | Why It Matters |
|-------------------|-------------------|----------------|
| **Channel management** | Middleware (FMOD, Wwise) | Voice limiting, priorities |
| **Synthesis programming** | Procedural audio | Understanding sound generation |
